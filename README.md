# 🗂️ Умный органайзер файлов

Автоматический скрипт для сортировки файлов по папкам в зависимости от их типа. Экономит время на ручную организацию загрузок и рабочих директорий.

## ✨ Возможности

- **Автоматическая сортировка**: Распределяет файлы по категориям (`images`, `documents`, `torrents` и др.).
- **Безопасный режим (`--dry-run`)**: Показывает, что будет сделано, без реального перемещения файлов.
- **Защита от дубликатов**: Три стратегии обработки:
  - `rename` — автоматически переименовывает (добавляет `_1`, `_2` и т.д.)
  - `skip` — пропускает файл при конфликте имён
  - `replace` — заменяет существующий файл
- **Отмена операций**: Ведётся журнал всех перемещений, можно отменить последнее действие или несколько (`--undo`, `--undo-count`).
- **Мониторинг**: Режим постоянного наблюдения за папкой с автоматической сортировкой новых файлов (`--monitor`).
- **Игнорирование системных файлов**: Не трогает скрытые файлы и сам скрипт.
- **Статистика**: Выводит отчет о количестве перемещенных файлов, созданных папок и удалённых пустых директорий.
- **Рекурсивная обработка**: Опциональная сортировка файлов во всех вложенных папках с очисткой пустых директорий.

## 🚀 Установка и использование

### Требования
- Python 3.6+
- Для GUI версии: библиотека `tkinter` (обычно встроена в Python, на Linux может потребоваться установка пакета `python3-tk`).
- Для режима мониторинга: опционально `watchdog` (`pip install watchdog`), при отсутствии используется polling-режим.

### Графический интерфейс (GUI)

Для удобного использования с визуальным управлением запустите GUI-версию:

```bash
python3 file_organizer_gui.py
```

**Возможности GUI:**
- **Выбор папки**: Кнопка для удобного выбора директории через проводник.
- **Рекурсивная обработка**: Опция для сортировки файлов во всех вложенных папках.
- **Тестовый режим**: Галочка для запуска без реального перемещения файлов (просмотр плана действий).
- **Логирование**: Окно с подробным отчетом о действиях в реальном времени.
- **Очистка**: Автоматическое удаление пустых папок после перемещения файлов (при рекурсии).
- **Прогресс-бар**: Визуальное отображение прогресса обработки.

### Консольный запуск

1. **Обычный режим** (файлы будут перемещены):
   ```bash
   python3 file_organizer.py /путь/к/папке
   ```
   *Если путь не указан, используется текущая директория.*

2. **Режим просмотра** (без изменений, только логирование):
   ```bash
   python3 file_organizer.py /путь/к/папке --dry-run
   ```

3. **Тихий режим** (минимум вывода):
   ```bash
   python3 file_organizer.py /путь/к/папке -q
   ```

4. **Без рекурсии** (только файлы в корне папки):
   ```bash
   python3 file_organizer.py /путь/к/папке --no-recursive
   ```

5. **Отмена последней операции**:
   ```bash
   python3 file_organizer.py /путь/к/папке --undo
   ```

6. **Отменить последние N перемещений**:
   ```bash
   python3 file_organizer.py /путь/к/папке --undo-count 5
   ```

7. **Режим мониторинга** (постоянное наблюдение за новыми файлами):
   ```bash
   python3 file_organizer.py /путь/к/папке --monitor
   ```

8. **Информация о доступных отменах**:
   ```bash
   python3 file_organizer.py /путь/к/папке --undo-info
   ```

### Примеры

Сортировка файлов в папке "Загрузки":
```bash
python3 file_organizer.py ~/Downloads
```

Проверка того, что будет сделано в текущей папке:
```bash
python3 file_organizer.py --dry-run
```

Отменить последние 3 перемещения:
```bash
python3 file_organizer.py ~/Downloads --undo-count 3
```

Запустить мониторинг папки "Загрузки":
```bash
python3 file_organizer.py ~/Downloads --monitor
```

## ⚙️ Как это работает

1. Скрипт сканирует все файлы в указанной директории (по умолчанию рекурсивно).
2. Определяет тип файла по расширению.
3. Создает целевую папку (например, `images`), если она не существует.
4. Перемещает файл в целевую папку.
   - При конфликте имён применяется стратегия из конфига (`rename` / `skip` / `replace`).
5. Все перемещения записываются в журнал `.organizer_undo.json`.
6. После завершения удаляются пустые папки (при рекурсивном режиме).
7. Выводится статистика по завершении.

## 🛡️ Безопасность

- Скрипт **не удаляет** файлы, только перемещает их.
- Игнорирует сам себя и скрытые файлы (начинающиеся с `.`).
- Режим `--dry-run` позволяет предварительно убедиться в корректности действий.
- **Защита от Path Traversal**: проверка, что все пути находятся внутри целевой директории.
- **Защита от TOCTOU**: атомарные операции с блокировками (threading.Lock).
- **Валидация путей**: запрет на использование `..`, абсолютных путей и symlink вне директории.
- **Защита от Command Injection**: безопасная работа с путями без subprocess/os.startfile.
- **Проверка symlink**: валидация назначения символических ссылок.
- **Защита от DoS**: лимит итераций (1000 вместо 100000).
- **SHA256 вместо MD5** для хеширования.
- **Лимит размера файла** (1GB по умолчанию).
- **Маскировка путей** в логах для приватности.
- **Защита системных директорий** от случайного удаления.
